## Prerequisites

Just Node.js 18+. That's it.

```bash
node --version
# Should show v18.0.0 or higher
```

If you don't have Node.js, install it:
- **macOS**: `brew install node`
- **Ubuntu/Debian**: `sudo apt install nodejs npm`
- **Windows**: Download from [nodejs.org](https://nodejs.org)

## One-Command Install (Recommended)

Installs Node.js (if needed), pnpm, Genie, AND Forge in one shot:

```bash
bash -c "$(curl -fsSL https://install.namastex.ai/get-genie.sh)"
```

**What this does:**
- Detects your OS
- Installs Node.js 18+ if missing
- Installs pnpm package manager
- Installs `automagik-genie` globally
- Installs `automagik-forge` (REQUIRED backend)
- Adds both to PATH

**Output you'll see:**
```
Detecting OS... ✓
Installing Node.js... ✓
Installing pnpm... ✓
Installing automagik-genie... ✓
Installing automagik-forge... ✓
Adding to PATH... ✓

Genie is installed!

Run 'genie' to get started.
```

## Manual Install

```bash
npm install -g automagik-genie
```

**IMPORTANT:** This installs both the Genie CLI AND Forge backend. Forge is not optional - it's the execution engine that actually runs your tasks.

The install creates:
- **genie** - CLI command
- **automagik-forge** - Backend server (runs automatically)

---

## Verify Installation

```bash
genie --version
# Should output: 2.5.11-rc.4 (or later)

which genie
# Should show path like: /usr/local/bin/genie

forge --version
# Should output: 2.5.11-rc.4 (or later)
```

If `genie --version` works but `which genie` fails, you have a PATH issue (see troubleshooting below).

---

## What Got Installed

- **genie** - Main CLI command
- **forge** - Backend server (bundled with genie)
- **.genie/** - Framework files (created on first run)

**Binary names:**
- Short: `genie`
- Full: `automagik-genie`

Both work the same. Use whichever you prefer.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command not found: genie">
    PATH issue. Add to `~/.bashrc` or `~/.zshrc`:

    ```bash
    export PATH="$PATH:$(npm config get prefix)/bin"
    ```

    Then reload:
    ```bash
    source ~/.bashrc  # or ~/.zshrc
    ```

    **Still not working?**
    ```bash
    # Find where npm installs global packages
    npm config get prefix
    # Example output: /usr/local

    # Check if genie is actually installed
    ls -la $(npm config get prefix)/bin/genie

    # If it exists but doesn't run, check permissions
    chmod +x $(npm config get prefix)/bin/genie
    ```
  </Accordion>

  <Accordion title="Permission denied">
    Use sudo for global installs:
    ```bash
    sudo npm install -g automagik-genie
    ```

    **OR** configure npm to use a user directory (recommended):
    ```bash
    mkdir ~/.npm-global
    npm config set prefix '~/.npm-global'
    export PATH=~/.npm-global/bin:$PATH

    # Add to ~/.bashrc or ~/.zshrc to make permanent
    echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
    ```

    Then install without sudo:
    ```bash
    npm install -g automagik-genie
    ```
  </Accordion>

  <Accordion title="npm ERR! code EACCES">
    You don't have write permissions to npm's global install directory.

    **Fix:**
    ```bash
    # Option 1: Use sudo (quick but less secure)
    sudo npm install -g automagik-genie

    # Option 2: Change npm's default directory (recommended)
    mkdir ~/.npm-global
    npm config set prefix '~/.npm-global'
    echo 'export PATH=~/.npm-global/bin:$PATH' >> ~/.bashrc
    source ~/.bashrc
    npm install -g automagik-genie
    ```
  </Accordion>

  <Accordion title="Wrong Node.js version">
    Genie requires Node.js 18+. Check your version:

    ```bash
    node --version
    # If < v18.0.0, upgrade:

    # macOS
    brew upgrade node

    # Ubuntu/Debian
    curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
    sudo apt-get install -y nodejs

    # Or use nvm (recommended)
    curl -o- https://raw.githubusercontent.com/nvm-sh/nvm/v0.39.0/install.sh | bash
    nvm install 18
    nvm use 18
    ```
  </Accordion>

  <Accordion title="Installation hangs or times out">
    Network issue or npm registry problem.

    ```bash
    # Try with verbose logging
    npm install -g automagik-genie --verbose

    # Or use a different registry
    npm install -g automagik-genie --registry https://registry.npmjs.org/

    # Check your internet connection
    ping registry.npmjs.org
    ```
  </Accordion>
</AccordionGroup>

## Environment Variables (Optional)

You can configure Genie via environment variables:

```bash
# Genie configuration
export GENIE_HOME="~/.genie"          # Where Genie stores data
export GENIE_LOG_LEVEL="info"        # Logging verbosity
export FORGE_PORT="8887"              # Forge backend port
export MCP_PORT="8885"                # MCP server port
```

Add these to `~/.bashrc` or `~/.zshrc` to make them permanent.

## Updating Genie

```bash
# Update to latest version
npm update -g automagik-genie

# Check for updates
npm outdated -g automagik-genie

# Install specific version
npm install -g automagik-genie@2.5.11-rc.4
```

---

## Uninstalling

```bash
# Remove Genie and Forge
npm uninstall -g automagik-genie

# Remove .genie directory (CAUTION: deletes your data!)
rm -rf ~/.genie

# Or just backup instead of deleting
mv ~/.genie ~/.genie.backup
```
