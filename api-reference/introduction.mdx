## Current API Access

While we build the comprehensive documentation, you can access the APIs directly:

### Omni REST API

```bash
# Start Omni server
omni serve --port 8000

# API available at http://localhost:8000
# Swagger docs at http://localhost:8000/docs
```

### Hive REST API

```bash
# Start Hive with API enabled
hive start --api --port 8001

# API available at http://localhost:8001
# Swagger docs at http://localhost:8001/docs
```

### Spark REST API

```bash
# Start Spark with API enabled
spark start --api --port 8002

# API available at http://localhost:8002
# Swagger docs at http://localhost:8002/docs
```

## MCP Server Access

Connect to MCP servers for AI agent integration:

### Genie MCP Server

```bash
# Start Genie MCP server
genie serve --mcp --port 3000
```

Add to Claude Desktop (`claude_desktop_config.json`):
```json
{
  "mcpServers": {
    "genie": {
      "command": "genie",
      "args": ["serve", "--mcp"]
    }
  }
}
```

### Omni MCP Integration

```bash
# Start Omni with MCP enabled
omni serve --mcp --port 3001
```

Add to Claude Desktop:
```json
{
  "mcpServers": {
    "omni": {
      "command": "omni",
      "args": ["serve", "--mcp"]
    }
  }
}
```

### Tools MCP Server

```bash
# Serve generated tools via MCP
uvx automagik-tools serve --tools github,slack
```

Add to Claude Desktop:
```json
{
  "mcpServers": {
    "automagik-tools": {
      "command": "uvx",
      "args": ["automagik-tools", "serve", "--tools", "github,slack"]
    }
  }
}
```

## Example: Using the Omni API

Here's a quick example of sending a WhatsApp message via the REST API:

```bash
# Send a message
curl -X POST http://localhost:8000/api/v1/messages \
  -H "Content-Type: application/json" \
  -d '{
    "instance": "my-whatsapp",
    "to": "+1234567890",
    "message": "Hello from Automagik!"
  }'
```

```python
# Python example
import requests

response = requests.post(
    "http://localhost:8000/api/v1/messages",
    json={
        "instance": "my-whatsapp",
        "to": "+1234567890",
        "message": "Hello from Automagik!"
    }
)

print(response.json())
```

```javascript
// JavaScript example
const response = await fetch('http://localhost:8000/api/v1/messages', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    instance: 'my-whatsapp',
    to: '+1234567890',
    message: 'Hello from Automagik!'
  })
});

const data = await response.json();
console.log(data);
```

## Authentication

Most APIs support multiple authentication methods:

<AccordionGroup>
  <Accordion title="API Key Authentication">
    ```bash
    curl -X GET http://localhost:8000/api/v1/instances \
      -H "X-API-Key: your-api-key"
    ```
  </Accordion>

  <Accordion title="Bearer Token Authentication">
    ```bash
    curl -X GET http://localhost:8000/api/v1/instances \
      -H "Authorization: Bearer your-token"
    ```
  </Accordion>

  <Accordion title="MCP Authentication">
    MCP servers use stdio-based communication - no HTTP auth required.
  </Accordion>
</AccordionGroup>
