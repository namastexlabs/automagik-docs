## Setup and Installation

<AccordionGroup>
  <Accordion title="What are the minimum system requirements?">
    **Hardware**:
    - CPU: 2+ cores recommended
    - RAM: 2GB minimum, 4GB+ recommended
    - Storage: 1GB for application, 5GB+ for logs/database
    - Network: Stable internet connection

    **Software**:
    - Python 3.12+ (3.11 supported but 3.12+ recommended)
    - Node.js 20+ (for Evolution API - WhatsApp)
    - PostgreSQL 13+ (optional, SQLite works for development)
    - Git (for installation)

    **Operating Systems**:
    - Linux (Ubuntu 20.04+, Debian 11+, RHEL 8+)
    - macOS 11+ (Big Sur and later)
    - Windows 10/11 with WSL2
  </Accordion>

  <Accordion title="Do I need Docker to run OMNI?">
    **No, Docker is not required.**

    Automagik OMNI can run directly on your system using:
    - Python virtual environment (via `uv`)
    - PM2 process manager
    - systemd service (Linux)

    Docker is **optional** and can be used if you prefer containerized deployments, but the default installation uses native processes for:
    - Easier debugging
    - Better performance
    - Simpler development workflow
    - No Docker overhead

    Evolution API (WhatsApp) also runs natively without Docker using SQLite database.
  </Accordion>

  <Accordion title="Can I use pip instead of uv?">
    **Yes, but uv is recommended.**

    Using pip:
    ```bash
    # Create virtual environment
    python -m venv venv
    source venv/bin/activate

    # Install dependencies
    pip install -r requirements.txt

    # Run application
    python -m src.api.app
    ```

    Why uv is better:
    - **10-100x faster** than pip
    - Automatic virtual environment management
    - Better dependency resolution
    - Built-in caching
    - Single tool for all Python package operations

    Install uv:
    ```bash
    curl -LsSf https://astral.sh/uv/install.sh | sh
    ```
  </Accordion>

  <Accordion title="How do I upgrade to the latest version?">
    ```bash
    # Stop services
    pm2 stop all

    # Backup database
    cp data/automagik-omni.db data/automagik-omni.db.backup

    # Pull latest code
    git pull origin main

    # Update submodules (Evolution API)
    git submodule update --remote

    # Update dependencies
    make install

    # Restart services
    pm2 restart all

    # Verify version
    curl http://localhost:8882/health | jq
    ```

    **Important**: Always backup your database before upgrading!
  </Accordion>

  <Accordion title="Where are configuration files located?">
    **Main configuration**:
    - `.env` - Environment variables (API keys, ports, database)
    - `ecosystem.config.js` - PM2 process configuration
    - `data/automagik-omni.db` - SQLite database (if using SQLite)

    **Evolution API configuration** (WhatsApp):
    - `resources/evolution-api/.env` - Evolution API settings
    - `resources/evolution-api/instances/` - WhatsApp session data

    **Logs**:
    - `logs/` - Application logs (if file logging enabled)
    - PM2 logs: `~/.pm2/logs/`

    **See also**: [Configuration Reference](/omni/config/environment-variables)
  </Accordion>
</AccordionGroup>

## Multi-Tenancy

<AccordionGroup>
  <Accordion title="What is multi-tenancy in OMNI?">
    **Multi-tenancy allows running multiple independent instances** from a single OMNI deployment.

    **Use Cases**:
    - Different WhatsApp numbers for different clients
    - Separate bots for sales, support, marketing
    - Testing vs production environments
    - Multiple Discord bots from one server
    - Per-customer isolated instances

    **Features**:
    - Complete data isolation between instances
    - Independent configurations (API keys, agents)
    - Separate webhook endpoints
    - Individual trace/analytics per instance

    Example:
    ```bash
    # Create multiple instances
    curl -X POST http://localhost:8882/api/v1/instances \
      -d '{"name": "sales-bot", ...}'

    curl -X POST http://localhost:8882/api/v1/instances \
      -d '{"name": "support-bot", ...}'

    # Each has unique webhook
    # /webhook/evolution/sales-bot
    # /webhook/evolution/support-bot
    ```

    [Multi-Tenancy Guide](/omni/concepts/multi-tenancy)
  </Accordion>

  <Accordion title="How many instances can I run?">
    **Technically unlimited**, but practical limits depend on resources.

    **Tested Scale**:
    - Single server: 100+ instances tested
    - Each instance: Handles 100-1000 messages/hour
    - Total throughput: 10,000+ messages/hour

    **Resource Requirements per Instance**:
    - Memory: ~50MB (WhatsApp), ~100MB (Discord)
    - CPU: Minimal when idle, spikes during message processing
    - Storage: ~10MB database + logs

    **Scaling Recommendations**:
    - **1-10 instances**: Single server, SQLite database
    - **10-50 instances**: Single server, PostgreSQL
    - **50-100 instances**: Dedicated server, PostgreSQL, connection pooling
    - **100+ instances**: Multiple servers, load balancer, PostgreSQL cluster

    [Scaling Guide](/omni/advanced/scaling-production)
  </Accordion>

  <Accordion title="Can instances share the same AI agent?">
    **Yes!** Multiple instances can use the same agent API.

    **Common Pattern**:
    ```bash
    # All instances point to same agent
    # Instance 1: sales-bot
    {
      "agent_api_url": "http://localhost:8886",
      "default_agent": "sales-agent"
    }

    # Instance 2: support-bot
    {
      "agent_api_url": "http://localhost:8886",
      "default_agent": "support-agent"
    }

    # Same agent API, different agent names
    # Agent can route based on instance or agent name
    ```

    **Benefits**:
    - Centralized agent management
    - Shared knowledge base
    - Easier updates
    - Cost optimization (single API deployment)

    **Considerations**:
    - Agent must handle concurrent requests
    - Implement per-instance context if needed
    - Monitor agent API load
  </Accordion>

    2. Pull latest: `git pull origin main`

    4. Restart: `pm2 restart all`
    5. Default instance auto-created from `.env`
    6. Create additional instances via API

    **No breaking changes** - existing setup continues to work!

    **Legacy webhook** still works:
    - `/webhook/evolution` ‚Üí Routes to default instance
    - New webhook: `/webhook/evolution/{instance}` ‚Üí Routes to specific instance

    [Migration Guide](/omni/installation#upgrading)

## Performance and Scaling

<AccordionGroup>
  <Accordion title="What is the message throughput capacity?">

    - **Outgoing**: 60 messages/minute (WhatsApp best practice)

    **OMNI Server (100 instances)**:
    - **Total throughput**: 10,000+ messages/hour

    - **Response time**: < 2 seconds (average)

    **Bottlenecks**:
    1. **AI Agent** - Slowest component (1-3 seconds)

    **Optimization**:
    - Use PostgreSQL for 50+ instances
    - Implement response caching
    - Use faster AI models
    - Enable message batching

    [Performance Tuning](/omni/advanced/scaling-production#performance-optimization)
  </Accordion>

  <Accordion title="Should I use SQLite or PostgreSQL?">
    **Use SQLite for**:
    - Development and testing
    - 1-10 instances
    - Small deployments (< 1000 messages/day)
    - Single server setups
    - Quick prototyping

    **Use PostgreSQL for**:
    - Production deployments
    - 10+ instances
    - High message volume (> 5000/day)
    - Multi-server setups
    - Better concurrency handling
    - Advanced analytics

    **Migration from SQLite to PostgreSQL**:
    ```bash
    # 1. Export SQLite data
    sqlite3 data/automagik-omni.db .dump > backup.sql

    # 2. Create PostgreSQL database
    createdb automagik_omni

    # 3. Update .env
    AUTOMAGIK_OMNI_DATABASE_URL=postgresql://user:pass@localhost/automagik_omni

    # 4. Import data (manual mapping needed)
    # Or recreate instances via API

    # 5. Restart
    pm2 restart all
    ```

    [Database Guide](/omni/config/database)
  </Accordion>

  <Accordion title="How do I reduce response latency?">
    **Common Causes of Slow Responses**:

    1. **Slow AI Agent** (most common)
       - Use faster models (Sonnet 4.5 vs GPT-4)
       - Reduce context length
       - Implement streaming (Automagik Hive)
       - Cache common responses

    2. **Network Latency**
       - Host services in same data center
       - Use low-latency cloud regions
       - Enable HTTP/2

    3. **Database Queries**
       - Migrate to PostgreSQL
       - Add database indexes
       - Enable connection pooling

    4. **Processing Overhead**
       - Disable tracing (if not needed)
       - Optimize webhook handling
       - Use message batching

    **Quick Wins**:
    ```bash
    # Increase agent timeout
    curl -X PATCH http://localhost:8882/api/v1/instances/my-bot \
      -d '{"agent_timeout": 120}'

    # Disable sensitive data in traces (reduces size)
    echo "AUTOMAGIK_OMNI_TRACE_INCLUDE_SENSITIVE=false" >> .env

    # Use connection pooling (PostgreSQL)
    AUTOMAGIK_OMNI_DATABASE_URL=postgresql://user:pass@localhost/db?pool_size=20
    ```

    [Performance Guide](/omni/advanced/scaling-production#latency-optimization)
  </Accordion>

  <Accordion title="Are messages stored or logged?">

    - Message metadata (timestamp, phone, instance)
    - Message content (incoming and outgoing)
    - Agent requests and responses
    - Processing metrics
    - Error logs

    **Storage Duration**:
    - Default: 30 days
    - Configurable: `AUTOMAGIK_OMNI_TRACE_RETENTION_DAYS`

    **Disable Tracing**:

    **Privacy Considerations**:
    - Messages contain user data
    - Ensure compliance with GDPR/CCPA
    - Implement data deletion policies
    - Secure database access
    - Consider encryption at rest

    **Cleanup**:
    ```bash
    # Delete traces older than 7 days
    curl -X POST http://localhost:8882/api/v1/traces/cleanup \
      -d '{"days_old": 7, "dry_run": false}'
    ```

## Troubleshooting

<AccordionGroup>
  <Accordion title="Why isn't my bot responding to messages?">
    **Quick Diagnostic**:
    ```bash
    # 1. Check services running
    pm2 status

    # 2. Check health
    curl http://localhost:8882/health

    # 3. Check instance exists
    curl -H "x-api-key: YOUR_KEY" \
      http://localhost:8882/api/v1/instances

    # 4. Check recent traces
    curl -H "x-api-key: YOUR_KEY" \
      "http://localhost:8882/api/v1/traces?limit=5"

    # 5. Check logs
    pm2 logs omni-api --lines 50
    ```

    **Common Causes**:
    - [ ] Evolution API not running
    - [ ] Webhook not configured
    - [ ] Instance not found
    - [ ] Agent API not responding
    - [ ] WhatsApp disconnected
    - [ ] Incorrect API keys

    [Full Troubleshooting Guide](/omni/troubleshooting/common-issues)
  </Accordion>

  <Accordion title="Where can I find logs?">
    **PM2 Logs**:
    ```bash
    # All services
    pm2 logs

    # Specific service
    pm2 logs omni-api
    pm2 logs evolution-api
    pm2 logs automagik-omni-discord

    # Error logs only
    pm2 logs --err

    # Last 100 lines
    pm2 logs --lines 100

    # Real-time tail
    pm2 logs --lines 0
    ```

    **File Logs** (if configured):
    ```bash
    # Application logs
    tail -f logs/omni.log

    # PM2 logs location
    ~/.pm2/logs/
    ```

    **Enable Debug Logging**:
    ```bash
    # Edit .env
    LOG_LEVEL=DEBUG

    # Restart
    pm2 restart all
    ```

    **Log Locations**:
    - PM2: `~/.pm2/logs/`
    - Application: `logs/` (if file logging enabled)
    - Evolution API: `resources/evolution-api/logs/`
  </Accordion>

  <Accordion title="How do I reset everything and start fresh?">
    **Complete Reset** (‚ö†Ô∏è Deletes all data):
    ```bash
    # 1. Stop all services
    pm2 stop all
    pm2 delete all

    # 2. Backup database (optional)
    cp data/automagik-omni.db data/backup-$(date +%Y%m%d).db

    # 3. Remove data
    rm -rf data/*.db
    rm -rf logs/*
    rm -rf /automagik-omni/sockets/*

    # 4. Remove Evolution API sessions
    rm -rf resources/evolution-api/instances/*

    # 5. Reinstall
    make install

    # 6. Start fresh
    pm2 start ecosystem.config.js

    # 7. Recreate instances
    # Use API or MCP to create instances
    ```

    **Partial Reset** (keep configuration):
    ```bash
    # Just reset database
    rm data/automagik-omni.db
    pm2 restart omni-api

    # Re-initialize from .env
    # Default instance will be created
    ```
  </Accordion>

  <Accordion title="What are the infrastructure costs?">
    **Typical Monthly Costs**:

    **Small Deployment** (1-5 instances, < 10K messages/month):
    - VPS: $5-10/month (DigitalOcean, Hetzner)
    - Database: $0 (SQLite) or $7 (managed PostgreSQL)
    - AI API: $10-50 (depending on usage)
    - **Total: $15-70/month**

    **Medium Deployment** (10-50 instances, 100K messages/month):
    - VPS: $20-40/month (4GB RAM, 2 CPU)
    - Database: $15-30 (managed PostgreSQL)
    - AI API: $100-500
    - **Total: $135-570/month**

    **Large Deployment** (100+ instances, 1M+ messages/month):
    - Servers: $100-500/month (multiple instances)
    - Database: $50-200 (PostgreSQL cluster)
    - AI API: $500-5000+
    - Load balancer: $10-50
    - **Total: $660-5750+/month**

    **Cost Optimization**:
    - Use cheaper AI models (Llama, Mistral)
    - Implement response caching
    - Use spot instances (AWS, GCP)
    - Self-host AI models (reduce API costs 90%)
  </Accordion>

  <Accordion title="How much does WhatsApp integration cost?">
    **Option 1: Evolution API + Regular WhatsApp** (what OMNI uses):
    - **Cost**: FREE
    - **Setup**: Free phone number
    - **Limitations**:
      - Not "official" WhatsApp Business API
      - Risk of WhatsApp bans if misused
      - Rate limits apply
    - **Best for**: Personal use, small businesses, testing

    **Option 2: Official WhatsApp Business API**:
    - **Cost**: $0.005-0.10 per message (varies by country)
    - **Setup**: Business verification required
    - **Benefits**:
      - Official API, no ban risk
      - Higher rate limits
      - Business features (catalog, payments)
      - Green verified badge
    - **Best for**: Large businesses, enterprises

    **Example Costs** (Official API):
    - 1,000 messages/month: $5-100
    - 10,000 messages/month: $50-1000
    - 100,000 messages/month: $500-10000
